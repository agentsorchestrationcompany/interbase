# Security Policy

## Supported Versions

This repository is pre-1.0 and does not currently publish a long-term support
matrix. Security fixes are expected to land on the default branch and in the
next public release.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use one of these private channels instead:

1. GitHub private vulnerability reporting / repository security advisories for
   `agentsorchestrationcompany/interbase`, if enabled.
2. If private reporting is not available, contact the repository owners through
   the GitHub organization and share the report privately.

Please include, when possible:

- affected package, file path, or feature area
- impact summary
- steps to reproduce
- proof of concept or logs, if safe to share
- suggested mitigation, if known

## Disclosure Process

Maintainers will acknowledge receipt, triage the issue, assess severity, and
coordinate remediation.

Please avoid public disclosure until maintainers have had a reasonable
opportunity to investigate and release a fix.

## Scope

This policy applies to this public repository, including the CLI wrapper,
CLI host workspace, SDK, and public extension packages.
